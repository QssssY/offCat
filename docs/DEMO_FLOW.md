# offcat 演示流程

本文档给出一条可按顺序执行的演示路线。更详细的准备项见 [演示指南](./DEMO_GUIDE.md)。

## 1. 用户端基础链路

1. 打开 `/`，展示首页和公共入口。
2. 打开 `/version-logs`，展示公开版本日志。
3. 打开 `/login`，注册普通用户。
4. 登录后进入 `/dashboard`，确认用户信息、会员状态和额度展示。
5. 打开 `/settings`，展示账号、安全、偏好、自定义 AI、通知、数据管理和外观设置。

验收点：

- 未登录只能访问公共页。
- 登录后用户端受保护路由可访问。
- 自定义 AI 与 TTS 配置页面不暴露明文密钥。

## 2. 简历诊断链路

1. 打开 `/resume/upload`。
2. 上传 PDF/DOC/DOCX 简历。
3. 创建诊断任务后进入 `/resume/result/:taskId`。
4. 观察任务阶段进度和最终诊断结果。
5. 打开 `/resume/history`，确认历史记录存在。
6. 在结果页触发 JD 匹配和 AI 润色。
7. 将润色结果进入模板编辑或导出链路。

验收点：

- 上传接口使用 `multipart/form-data`。
- 任务只允许当前用户访问。
- 失败任务可重试。
- PDF 导出需要后端 Chrome 配置。

## 3. 模板编辑链路

1. 打开 `/templates`。
2. 按行业筛选模板。
3. 进入 `/templates/editor/:templateId`。
4. 修改基础信息、经历、技能、项目、证书等奖项段落。
5. 预览模板并导出 PDF、PNG 或 DOCX。

验收点：

- 模板使用会走 `/api/template/use`。
- 导出内容应与预览一致。
- 富文本内容需经过前端和后端安全处理。

## 4. 模拟面试链路

1. 打开 `/interview/entry`。
2. 选择岗位、难度、面试模式、反馈模式和交互方式。
3. 可选输入 JD、关联简历任务或复用最近一次 JD 匹配。
4. 创建会话后进入 `/interview/session/:sessionId`。
5. 发送消息，观察 SSE 流式回复。
6. 如选择语音面试，验证 TTS 能力和播放。
7. 结束面试，进入 `/interview/report/:sessionId`。
8. 打开 `/interview/history`，查看历史列表。

验收点：

- 流式消息应正常返回 `content` 与 `done` 事件。
- 结束后不能继续发送消息。
- 报告包含综合评分和六维评分。
- 历史删除只能影响当前用户数据。

## 5. Offer、成长与通知

1. 打开 `/offer`，执行薪资谈判模拟。
2. 生成薪资谈判话术脚本。
3. 打开 `/growth`，查看成长概览和面试雷达。
4. 打开 `/notifications`，验证通知列表、未读数、已读和删除。
5. 观察顶部通知入口的未读状态变化。

验收点：

- Offer 助手请求受登录和限流保护。
- 成长中心数据来自当前用户。
- 通知 SSE 可实时推送或按轮询状态展示。

## 6. 社区链路

1. 打开 `/community`。
2. 创建帖子，可附带图片或面试报告链接。
3. 打开帖子详情，发表评论和回复。
4. 对帖子执行点赞、收藏。
5. 打开 `/community/my`，查看我的活动和收到的互动。

验收点：

- 社区图片上传走 `/api/community/images/upload`。
- 内容审核状态由后端维护。
- 互动未读数与我的活动页面一致。

## 7. 管理后台链路

1. 打开 `/admin/login`，使用管理员账号登录。
2. 打开 `/admin/dashboard`，查看数据看板。
3. 打开 `/admin/monitor`，查看业务监控。
4. 管理岗位、Prompt、AI 引擎和 TTS 配置。
5. 管理用户权益、封禁、配额和消耗记录。
6. 管理会员套餐、订单、通知和版本日志。
7. 审核社区帖子与评论。
8. 查看反馈、审计日志和自定义 AI 用量。

验收点：

- 非管理员不能访问 `/admin/**`。
- 管理端 Token 与用户端 Token 相互独立。
- 批量操作、发布操作和删除操作均有确认与错误反馈。

## 8. 演示收尾

1. 运行后端测试或编译检查。
2. 运行前端测试或构建检查。
3. 检查 Git 状态，确认没有敏感配置文件被加入提交。
4. 根据本次演示模式记录是否使用真实 AI、TTS、OSS。
