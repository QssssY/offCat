# 项目演示流程

## 1. 演示目标
本次演示用于展示“智能模拟面试与简历诊断系统”后端核心模块已经完成，能够支持用户认证、额度控制、简历诊断任务链路、模拟面试任务链路以及基础历史查询能力。

## 2. 演示前准备

### 2.1 环境准备
确保以下服务已正常启动：
- MySQL
- Redis
- RabbitMQ
- Spring Boot 后端服务

### 2.2 数据准备
确保数据库已执行初始化脚本：
- `db/schema.sql`

### 2.3 演示说明
当前版本已完成核心业务闭环：
- 认证模块
- 额度模块
- 简历诊断基础链路
- 模拟面试基础链路

说明：
- 当前简历诊断结果和面试回复/评分可采用 mock/stub 方式生成
- 真实大模型接入属于增强项，当前版本未强依赖 API Key

---

## 3. 演示流程

## 3.1 用户注册
### 接口
- `POST /api/auth/register`

### 演示内容
提交用户名和密码，完成普通用户注册。

### 预期结果
- 返回注册成功
- 数据库 `sys_user` 中新增用户记录
- 数据库 `user_quota` 中自动初始化额度记录

---

## 3.2 用户登录
### 接口
- `POST /api/auth/login`

### 演示内容
使用已注册账号登录系统。

### 预期结果
- 返回 JWT Token
- 后续接口调用可携带该 Token 进行鉴权

---

## 3.3 获取当前用户信息
### 接口
- `GET /api/auth/me`

### 演示内容
使用登录后获取的 Token 查询当前用户信息。

### 预期结果
- 返回当前登录用户的基础信息
- 证明鉴权链路可用

---

## 3.4 简历诊断任务创建
### 接口
- 简历上传/任务创建相关接口

### 演示内容
提交简历文件地址或上传请求，创建简历诊断任务。

### 预期结果
- 成功创建任务
- 返回任务 ID
- 数据库 `resume_diagnosis_task` 中新增任务记录
- 任务初始状态为排队中或处理中

---

## 3.5 简历诊断任务处理结果查询
### 接口
- 简历任务状态查询接口
- 简历历史记录查询接口

### 演示内容
查询任务状态与任务结果。

### 预期结果
- 能看到任务状态流转
- 能看到结构化诊断结果或 mock 结果
- 能看到历史任务列表

---

## 3.6 创建模拟面试会话
### 接口
- 模拟面试会话创建接口

### 演示内容
创建一场新的模拟面试，指定岗位与难度。

### 预期结果
- 返回会话 ID
- 数据库 `interview_session` 中新增会话记录

---

## 3.7 发送面试消息
### 接口
- 模拟面试消息发送接口

### 演示内容
发送用户回答内容，触发 mock 面试官回复。

### 预期结果
- 数据库 `interview_chat_log` 中新增消息记录
- 返回 mock 面试官回复内容
- 会话链路正常流转

---

## 3.8 结束面试并生成结果
### 接口
- 面试结束接口
- 面试详情查询接口

### 演示内容
结束当前面试会话并查询会话详情。

### 预期结果
- 会话状态更新为已结束
- 生成 mock 综合评分
- 生成 mock 评价报告
- 查询详情时可看到完整记录

---

## 3.9 查看历史面试记录
### 接口
- 面试历史列表接口
- 面试详情接口

### 演示内容
查询当前用户历史面试记录与详情。

### 预期结果
- 返回历史列表
- 可查看单次会话详情与评价报告
